# 七咲式ヒアリング質問

Use this reference when the user has not provided enough project information or when the first answers need deeper clarification.

## Minimum Questions

Ask these first when starting from little context:

1. どんな業種・サービスのホームページですか？
2. このサイトを見た人に、最終的にどんな行動をしてほしいですか？
3. 一番来てほしいお客様はどんな人ですか？
4. その人は申し込む前に、何に悩んでいますか？
5. 競合ではなく、あなたを選ぶ理由を3つ挙げるなら何ですか？
6. サイトで出したい印象と、避けたい印象は何ですか？
7. 掲載できる実績、事例、お客様の声、写真、資格はありますか？

## Deep Questions

Use these after the initial answers.

### Visitor and Problem

- そのお客様は、いつ・どんな状況でこのサービスを探しますか？
- 申し込む直前に感じる一番大きな不安は何ですか？
- 過去に他の方法を試して失敗した経験はありますか？
- このサービスを受けない場合、どんな困りごとが続きますか？

### Offer

- 提供するサービス内容を具体的に教えてください。
- 初回相談、体験、無料資料、予約など、最初の入口は何ですか？
- 料金は掲載しますか？掲載しない場合、問い合わせ前の不安をどう減らしますか？
- 申し込み後の流れは何ステップですか？

### Differentiation

- 価格以外で勝てるポイントは何ですか？
- お客様からよく褒められる点は何ですか？
- 逆に誤解されたくないことは何ですか？
- 競合サイトと比べて、絶対に同じにしたくない点は何ですか？

### Proof

- 掲載できる実績や数値はありますか？
- お客様の声は掲載許可がありますか？
- Before / After を使う場合、表現上の注意はありますか？
- 代表者プロフィールや資格はありますか？

### Design

- 高級感、親しみやすさ、専門性、スピード感など、どの印象を優先しますか？
- 参考にしたいサイトはありますか？
- 避けたい色、雰囲気、表現はありますか？
- スマホで最初に見せたい情報は何ですか？

## Asking Style

Ask 3 to 5 questions at a time.

When the user's answer is vague, give examples and ask them to choose.

When the user says "おまかせ", make a reasonable assumption and clearly mark it as an assumption.
