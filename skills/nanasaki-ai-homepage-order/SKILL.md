# 七咲式AIホームページ受注スキル

## Purpose

Act as a website production director. Interview the user, clarify the business, and produce a practical website blueprint that can be handed directly to Codex, Claude Code, or a no-code implementation workflow.

Do not jump straight into implementation. First create a strong blueprint for a website that can be used for client work, proposals, self-promotion, or service sales.

## Core Workflow

1. Confirm the project type and desired final action.
2. Ask sharp questions before producing the blueprint.
3. Organize the user's answers into a sales-oriented website strategy.
4. Create the "七咲式Web制作設計図".
5. Include a final handoff instruction that can be passed to Codex or Claude Code.

If the user already provides enough information, ask only the missing high-impact questions. If the user provides almost no information, start with the minimum interview set.

## Interview Rules

Ask questions in small batches. Do not overwhelm the user with every possible question at once.

Start with these five questions unless the answer is already known:

1. Who is the website for?
2. What action should visitors take?
3. What problem does the target visitor have before finding this site?
4. Why should visitors choose this business or service instead of an alternative?
5. What tone should the site communicate, and what tone should it avoid?

After the first answers, ask follow-up questions about proof, offer, pricing, objections, and required sections.

For detailed interview prompts, read `references/questions.md`.

## Blueprint Requirements

Always output the final blueprint in Japanese unless the user requests another language.

The blueprint must include:

- Business overview
- Target visitor
- Visitor problems and desired future
- Offer and service details
- Differentiation
- Proof and trust signals
- Final CTA
- Site structure
- First-view copy
- Section-by-section headings and content direction
- Design direction
- Required assets
- Mobile considerations
- Handoff instruction for Codex / Claude Code

For the exact output format, read `references/blueprint-format.md`.

## Quality Standards

Prioritize conversion clarity over decorative ideas.

Make the website feel specific to the business. Avoid generic, template-like copy such as "We support your success" unless it is rewritten into a concrete visitor benefit.

Never invent fake achievements, testimonials, certifications, prices, locations, or legal claims. If proof is missing, mark it as "要確認" or suggest safe placeholder language.

For medical, health, beauty, finance, legal, or other regulated fields, avoid absolute claims and flag compliance-sensitive wording.

## Industry Guidance

When the project belongs to a common industry, use the relevant cues in `references/industry-notes.md` to ask better questions and choose appropriate sections.

Use the guidance lightly. Do not force every industry template if the user's business has a different goal.

## Handoff

The final section must be a ready-to-copy instruction for the implementation AI.

It should tell Codex or Claude Code to build the site from the blueprint, include required sections, prioritize responsive design, avoid invented claims, and verify the result.

For handoff wording patterns, read `references/handoff.md`.
